Numerical Methods & Discretisation

How PDEs become algebraic equations — the core trick behind all CFD, and what discretisation choices mean for accuracy.

Finite difference approximations

\[ \frac{\partial u}{\partial x}\bigg|_i\approx\frac{u_{i+1}-u_i}{\Delta x}\;\text{(forward, }O(\Delta x)\text{)},\quad\frac{u_{i+1}-u_{i-1}}{2\Delta x}\;\text{(central, }O(\Delta x^2)\text{)} \]
Truncation error: Forward difference: 1st order (\(O(\Delta x)\)). Central difference: 2nd order (\(O(\Delta x^2)\)) — more accurate for the same mesh size. Fluent uses finite volume but the accuracy concepts are identical.
\[ \frac{\partial^2u}{\partial x^2}\bigg|_i\approx\frac{u_{i+1}-2u_i+u_{i-1}}{\Delta x^2}\quad O(\Delta x^2) \]
Second-order central difference: Used for viscous/diffusion terms. Has \(O(\Delta x^2)\) accuracy. In Fluent: choose "Second Order Upwind" for convective terms and "Second Order" for pressure to get 2nd-order accuracy overall.

Time marching — transient simulations

\[ \frac{u^{n+1}-u^n}{\Delta t}=RHS^n\;\text{(explicit)},\quad \frac{u^{n+1}-u^n}{\Delta t}=RHS^{n+1}\;\text{(implicit)} \]
Time integration: Explicit: updates depend on current values — conditionally stable, requires small \(\Delta t\) (CFL ≤ 1). Implicit: updates depend on new values — unconditionally stable, larger \(\Delta t\) allowed but requires solving a system of equations per step. Fluent transient uses implicit time integration by default.
\[ CFL=\frac{V\Delta t}{\Delta x}\leq 1\quad\text{(explicit stability)} \]
CFL (Courant-Friedrichs-Lewy) condition: For explicit methods, the information must not travel more than one cell per time step. Fluent's pseudo-transient solver adjusts this internally. For transient LES: target CFL ≈ 0.5–1 near walls.

Relaxation factors: SIMPLE typically uses 0.3 for pressure and 0.7 for momentum. If divergence occurs, reduce to 0.2/0.5. If convergence is too slow, try Coupled solver or SIMPLEC. Do not increase relaxation factors beyond defaults unless the simulation is very stable.

Numerical diffusion and its control

\[ \phi_{face} = \phi_{upwind} + \nabla\phi\cdot\vec{r}\quad\text{(2nd order reconstruction)} \]
Face value reconstruction: In first-order upwind, the face value = upstream cell centre value — this is equivalent to adding extra diffusion (numerical diffusion) proportional to the cell size. Second-order upwind reconstructs a gradient and extrapolates to the face — dramatically reducing numerical diffusion at the cost of potential oscillations at sharp gradients.
